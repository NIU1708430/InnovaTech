# Caso de Uso: Seleccionar y Realizar Pago

### Versión
1.0

### Fecha
30/04/2025

### Autores
Elías Pascual Paz

---

### Descripción
Este caso de uso describe cómo el cliente selecciona un método de pago y efectúa el pago del pedido. La aplicación permite pagar mediante tarjeta bancaria o PayPal. Se debe garantizar la seguridad de los datos financieros en todo momento.

---

### Actores
- Cliente  
- Sistema de Pago 

---

### Precondiciones
- El cliente ha iniciado sesión.  
- El cliente ha finalizado la selección de platos y confirmado el pedido.  
- El sistema ha calculado el total del pedido incluyendo el coste de envío.

---

### Flujo Principal
1. El cliente accede al proceso de pago tras confirmar su pedido.
2. El sistema muestra las opciones de pago disponibles: tarjeta bancaria y PayPal.
3. El cliente selecciona el método de pago deseado.
4. El sistema redirige al cliente al servicio correspondiente (pasarela bancaria o PayPal).
5. El cliente introduce sus datos de pago y confirma la transacción.
6. El sistema recibe la confirmación del pago y finaliza el proceso de pedido.


---

### Subflujos
- **Autenticarse en Sistema de Pago** 

---

### Flujos Alternativos
- **Pago rechazado o error en la transacción**:  
  El sistema muestra un mensaje de error y permite al cliente volver a intentar el pago o seleccionar otro método.

- **Cancelación del pago por parte del usuario**:  
  El sistema cancela el proceso de pago y vuelve a la pantalla de selección del método de pago.

---

### Postcondiciones
- El pedido queda registrado como pagado.
- El cliente recibe una confirmación del pedido.

---

### Requisitos No Funcionales
- Cumplimiento de estándares de seguridad para pagos.
- Uso de conexiones seguras (HTTPS).
- Integración con plataformas de pago seguras como PayPal.

---

### Prioridad
Alta

---

### Comentarios
Es fundamental proteger los datos sensibles del cliente durante la transacción para garantizar la confianza y cumplimiento legal.
